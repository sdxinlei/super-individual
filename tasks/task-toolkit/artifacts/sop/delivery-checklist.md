# 交付检查清单

> **目的：** 每次交付前逐项检查，确保质量
> **使用方式：** 复制到项目 Issue 或本地 Markdown，逐项打勾

---

## 一、代码检查

- [ ] 所有 `.env` / 密钥文件已加入 `.gitignore`
- [ ] 无硬编码密钥、Token、密码
- [ ] 代码注释清晰，无遗留的 `TODO` / `FIXME` / 调试日志
- [ ] 无死代码（被注释的无用代码、未使用的 import）
- [ ] 函数/变量命名规范（驼峰或下划线统一）
- [ ] 代码格式化（Prettier / ESLint / Ruff 等工具已跑过）
- [ ] Git 提交信息规范，无 `asdf` / `fix` 等无意义提交

---

## 二、测试检查

- [ ] 核心功能手动测试通过
- [ ] 边界条件测试（空输入、超长输入、特殊字符）
- [ ] 异常流程测试（网络断开、API 返回错误）
- [ ] 如果涉及表单：必填验证、格式校验均正常
- [ ] 如果涉及移动端：在 375px / 414px 宽度下测试过

---

## 三、文档检查

- [ ] 部署文档已编写（环境要求、安装步骤、启动命令）
- [ ] 如有 API：接口文档已附上请求/响应示例
- [ ] 如有环境变量：已列出所有变量及说明
- [ ] 客户操作手册（如需）已交付

---

## 四、部署检查

- [ ] 生产环境已部署并可访问
- [ ] 域名 / SSL 证书已配置且有效
- [ ] 环境变量已正确配置（区分开发/生产）
- [ ] 数据库迁移已执行
- [ ] 静态资源（图片/字体等）加载正常
- [ ] 控制台无报错（F12 检查）
- [ ] 响应式布局在常用设备上可用
- [ ] 已做基本性能检查（首屏加载 < 3s）

---

## 五、安全检查

- [ ] 无敏感信息泄露（API Key / 数据库密码 / 内部 URL）
- [ ] 用户输入已做 XSS 防护（输出转义）
- [ ] 接口权限控制正确
- [ ] HTTPS 已强制开启

---

## 六、交付确认

- [ ] 源代码已推送至客户仓库（或提供压缩包）
- [ ] 客户已收到部署地址并验证可用
- [ ] 客户已确认所有交付物清单
- [ ] 尾款已开具发票
- [ ] 售后通道已建立（微信 / 邮件 / 工单）
