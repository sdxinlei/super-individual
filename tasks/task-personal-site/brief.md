# Task: task-personal-site - 个人主页/作品集站点

## 背景
根据「超级个体计划-执行篇」Day 4 要求：用 GitHub Pages 搭建极简个人主页，展示作品 + 联系方式。现有 D:\超级个体\index.html 是一个占位页面（"超级个体实验室"），需要升级为真正的个人作品集主页。

## 输入
- 现有占位文件: D:\超级个体\index.html
- 计划要求: 极简个人主页，展示作品 + 联系方式
- 部署方式: GitHub Pages / 静态 HTML（纯前端，无需后端）

## 要求
1. 单页 HTML（可内嵌 CSS/JS），响应式，移动端适配
2. 包含：品牌标识（超级个体 × AI 双引擎）、作品展示区（至少 3 个卡片占位）、联系方式、GitHub 链接
3. 技术栈：纯 HTML + CSS + JS，零依赖，可直接在浏览器打开或部署到 GitHub Pages
4. 设计风格：科技感、简洁、深色主题（与现有 index.html 风格一致但更丰富）
5. 在底部加入 Google Analytics 占位（可后续配置 ID）

## 产出文件
- tasks/task-personal-site/artifacts/index.html: 新个人主页
- tasks/task-personal-site/artifacts/style.css: 独立样式（可选，也可内嵌）
- tasks/task-personal-site/result.md: 完成报告

## 注意事项
- 不要引入 npm/node_modules 依赖
- 所有文件必须是静态可部署
- 图标可用 SVG inline 或 Font Awesome CDN
