## 二、generate-covers.js — 生成脚本

### 逻辑正确性

#### extractCoverData 函数
- 按 `##`/`###` + "笔记"/"篇" 识别笔记章节，逻辑正确。
- 提取 `封面卡标题` / `封面卡副标题`：使用 `split('：')` 处理中英文冒号，覆盖了 markdown 中的实际格式，正确。
- 实际运行提取到 15 条数据，与 xiaohongshu-15.md 中的笔记数量一致。

#### BUG 发现：noteTitle 提取失效（低影响）
```js
const titleLine = lines.slice(Math.max(0, lines.indexOf(line)-5), lines.indexOf(line))
  .find(l => l.startsWith('标题'));
```
问题：markdown 中的标题行格式为 `**标题**：...`，以 `**` 开头，所以 `.startsWith('标题')` 不匹配。此变量仅用于 HTML `<title>` 标签（用户不可见），实际不影响封面的视觉内容。所有 15 张卡的 `<title>` 都回退为 `"效率工具"`。

#### 文件名生成
```js
data.title.slice(0, 8).replace(/[/\\?%*:|"<>]/g, '')
```
- 取前 8 个字符 + 剔除非法字符，15 个文件名无重复、无冲突。
- 注意：`slice(0,8)` 对 CJK 字符按字符数截取，导致部分文件名被截断（如 "副业第1年心态崩了" → "cover-06-副业第1年心态崩"），但不影响功能。

#### 构建流程
- 优先读取 xiaohongshu-15.md 提取数据，不足 15 条时使用 `defaultCovers` 回退。
- 运行 `node generate-covers.js` 一次通过，无报错，15 个文件全部生成。

#### 代码健壮性
- `lines.indexOf(line)` 会返回字符串在数组中的首次出现索引，若存在重复行则定位错误。当前 markdown 无重复行，不影响。
- 硬编码的 Windows 绝对路径（以 `d:/` 开头），在 Windows 下能运行，但跨平台兼容性较差。
