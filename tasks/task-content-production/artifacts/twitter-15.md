## Dev Tips (5)

### 6. BYOK Pattern

Best UX pattern nobody talks about. Users bring their API key = zero backend, zero data liability, zero signup friction. Pure frontend wins.

🎨 Diagram: Browser -> User API Key -> LLM

#WebDev #DevTips #FrontendArchitecture

### 7. Claude Code + Codex

My AI pairing: Claude Code as architect, Codex as executor. One brain, two hands. Review the plan first, then let Codex run. Zero context switches.

🎨 Terminal screenshot showing Claude Code + Codex workflow

#DevTips #ClaudeCode #AIPairProgramming

---

### 8. Component-First DX

Stop organizing by file type (components/, hooks/, utils/). Organize by feature instead. Every feature folder ships its own types, tests, and styles. Scales without pain.

🎨 Side-by-side: messy flat folder vs clean feature folders

#WebDev #React #CleanCode

---

### 9. API Key Tradeoffs

API Key pattern: great for prototypes, risky for scale. Users hit rate limits they don't understand, then blame your app. Always add a proxy layer before v2.

🎨 Screenshot of a rate limit error toast on a demo app

#DevTips #APIDesign #FrontendDev

---

### 10. Type Over Test

I ship faster with strict TypeScript than unit tests. Types catch 80% of bugs at write time. Tests catch the rest. If you skip types, your tests will be twice as long.

🎨 Editor screenshot showing strict TS config with red squigglies

#TypeScript #WebDev #DevTips
