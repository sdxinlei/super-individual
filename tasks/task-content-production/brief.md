# Task: content-production - 大规模内容生产（明早9点截止）

## 总目标
生产 50+ 条高质量社交媒体内容，每条可直接发布。Claude Code 和 Codex 互相检测，确保质量。

## 内容分配

### 即刻 (jike) — 20 条
主题矩阵（每个主题4条）：
1. **独立开发日常**：bug 故事、熬夜、上线瞬间、用户反馈
2. **工具推荐**：文案助手、小红书工厂、周报生成器（轮流推）
3. **AI 工作流**：Claude Code + Codex 双引擎怎么配合、Prompt 技巧
4. **创业思考**：超级个体理念、一人公司、不融资
5. **技术分享**：纯前端模式、API Key 模式、0 运维

每条格式：
```
【标题/一句话hook】

正文（80-150字，口语化，有故事感）

标签：#超级个体 #独立开发 #AI
```

### 小红书 — 15 篇
主题矩阵：
1. **AI效率工具**（3篇）：独立开发者/自媒体/学生各一篇
2. **副业赚钱**（3篇）：自由职业/AI副业/数字游民
3. **小红书运营**（3篇）：标题技巧/标签策略/排版
4. **工具种草**（3篇）：文案助手/小红书工厂/周报
5. **心态/成长**（3篇）：裸辞故事/30岁/自律

每篇格式：
```
标题：hook型标题，带数字/痛点/好奇

正文：800-1200字
- 开头：痛点/故事引入（100字）
- 中间：3-5个要点，分段，加 emoji
- 结尾：行动号召/评论区互动

标签：#标签1 #标签2 #标签3 #标签4 #标签5
```

**每个小红书笔记必须配一个 HTML 封面卡片**（视觉吸睛，深色科技风）

### X/Twitter — 15 条
- Building in Public（5条）
- Dev Tips（5条）
- Tool launches（5条）

格式：<200 chars，英文

## 视觉要求（小红书封面卡）
封面卡 HTML 标准：
- 尺寸：1080×1080px（1:1 方形，小红书标准）
- 深色科技风，靛蓝紫渐变
- 大标题醒目，30% 留白
- 右下角水印「超级个体」
- 用 CSS 实现，浏览器截图即用

## 质量红线
- ❌ 没有数据支撑的"很高效""非常好" — 必须具体（"提效3倍"/"3天完成"）
- ❌ 泛泛而谈 — 必须有故事/案例/步骤
- ❌ 机器人感 — 口语化，有个人风格
- ❌ 封面丑 — 必须好看
