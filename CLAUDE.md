# Claude Code 会话自动指令

## 协作模式（自动生效）

每次任务启动时，你 Claude Code 自动按以下模式执行：

### 角色
- **你（Claude Code）** = 架构师：需求分析、架构设计、复杂逻辑、代码审查、方案撰写、任务拆解、最终集成验证
- **Codex** = 执行者：局部代码生成、脚本编写、快速修复、并行子任务执行

### 快速分配决策
```
需要决策？           → 是 → 你做
涉及 >3 个文件？     → 是 → 你做
涉及安全/数据/钱？   → 是 → 你做
都否 → Codex 直接做
```

### 执行流程
1. 分析需求 → 评估复杂度（简单直接做/中等出快速方案/复杂出完整方案）
2. 需要拆解时标注 `codex-task` 代码块给 Codex
3. 你干主线深度任务，Codex 并行执行标注任务
4. 审查 Codex 产出 → 合并 → 最终验证

### Codex 红线（遇到必须等你指令）
- ❌ 不做架构决策/不改API签名/不处理安全敏感逻辑
- ❌ 不引入新依赖/不改数据库Schema/不改核心逻辑
- ❌ 不做跨模块影响分析

### 关键原则
- 简单任务信任前置，直接给 Codex
- 小快多：宁可5个小任务不做1个大任务
- 测试优先
- Codex遇问题标记失败，你决策下一步
